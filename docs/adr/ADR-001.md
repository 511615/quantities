# ADR-001: 数字资产量化研究平台基础架构

## 状态

已采纳

## 决策

- 采用 `Python 模块化单体`，以显式 contract 解耦 `data / features / datasets / models / training / backtest / agents`
- 配置体系采用 `Hydra + Pydantic`
- 研究运行以 `manifest + digest + seed` 固化可复现上下文
- 回测只消费 `PredictionFrame` 或 `SignalFrame` 等桥接对象，不读取训练内部状态
- Agent 仅通过受限 facade 与 artifact 交互，不直接操作核心内部对象

## 当前实现范围

- 已完成仓库骨架、配置分层、核心 contract、基线模型 registry、本地训练 runner、占位回测引擎、Agent 研究服务、CLI 和基础测试
- 尚未实现真实市场数据接入、复杂特征流水线、专业级撮合引擎、外部跟踪后端和执行接口

## 关键约束

- 标签只能在 `datasets` 层生成
- `FeatureViewRef` 必须声明 `as_of_time`
- 任何样本都必须满足 `available_time <= as_of_time`
- 训练只接受 `DatasetRef`
- 回测只接受 `PredictionFrame` 引用

## 后续扩展

- 在现有 registry 上接入树模型和深度模型
- 将 `file tracking` 替换或补充为 `MLflow` adapter
- 在 `backtest/engines` 下增加事件驱动引擎
- 在 `agents/tools` 下暴露结构化研究工具，而不是直连内部模块
